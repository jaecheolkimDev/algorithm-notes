## 1️⃣ 그래프 이론 (Graph Theory)

### 📘 정의
정점(Vertex)과 간선(Edge)으로 구성된 자료구조로, 사물이나 개체 간의 **연결 관계(Connection / Relation)** 를 표현함.

### 📗 주요 용어
- **정점(Vertex)** : 그래프의 기본 단위. 사람, 도시 등 개체를 의미.
- **간선(Edge)** : 두 정점을 연결하는 선. 관계나 경로를 표현.
- **방향 그래프(Directed Graph)** : 간선에 방향이 있음 (A → B ≠ B → A).
- **무방향 그래프(Undirected Graph)** : 간선에 방향이 없음 (A–B = B–A).
- **가중치(Weight)** : 간선에 부여된 거리, 비용, 시간 등의 값.
- **차수(Degree)** : 정점에 연결된 간선의 개수.
    - 방향 그래프에서는 진입차수(In-degree), 진출차수(Out-degree)로 구분.
- **인접(Adjacency)** : 두 정점이 간선으로 직접 연결되어 있는 상태.
- **경로(Path)** : 한 정점에서 다른 정점으로 이동하는 정점들의 순서.
- **사이클(Cycle)** : 한 정점에서 출발하여 다시 자기 자신으로 돌아오는 경로.
- **컴포넌트(Component)** : 연결된 정점들의 묶음 (연결 요소).
- **그래프 표현 방식**
    - **인접 행렬(Adjacency Matrix)** : 2차원 배열로 연결 여부를 표시.
    - **인접 리스트(Adjacency List)** : 각 정점이 연결된 정점 리스트로 저장.

## 2️⃣ 그래프 탐색 (Graph Traversal)

### 📘 정의
그래프의 모든 정점을 한 번씩 방문하는 과정.

### 📗 탐색 방식
- **DFS (Depth First Search, 깊이 우선 탐색)**
    - 한 경로를 끝까지 탐색한 뒤 돌아오는 방식.
    - **구현** : 스택(Stack) 또는 재귀(Recursion) 사용.
    - **특징** : 백트래킹(Backtracking) 가능 — 경로 탐색에 유리.

- **BFS (Breadth First Search, 너비 우선 탐색)**
    - 가까운 노드부터 차례로 탐색.
    - **구현** : 큐(Queue) 사용.
    - **특징** : 최단 거리(Shortest Path) 문제에 유용.

### 📙 관련 개념
- **방문 배열(Visited Array)** : 방문한 정점을 표시하여 중복 방지.
- **경로 추적(Path Tracking)** : 어떤 경로로 도달했는지 저장.
- **응용 예시** : 미로 탐색, 연결 요소 찾기, 최단 거리 구하기.

- 깊이우선탐색(DFS, Depth First Search) : 방문배열과 재귀를 통해서 구현한다.
    - DFS는 그래프를 탐색할 때 쓰는 알고리즘이며 어떤 노드부터 시작해 인접한 노드들을 재귀적으로 방문하며 방문한 정점은 다시 방문하지 않으며 각 분기마다 가능한 멀리 있는 노드까지
      탐색하는 알고리즘입니다.
    - 메모리를 덜 씀. 절단점 등 구할 수 있음. 코드가 좀 더 짧으며 완전탐색의 경우에 많이 씀.
    - 탐색과정에서 다시 돌아오는 등 중복 방문이 생길 수 있기 때문에 시간초과가 발생하기 쉬워서 BFS를 권장한다.
- 너비우선탐색(BFS, Breadth First Search) : 큐를 통해서 구현한다.
    - BFS는 그래프를 탐색하는 알고리즘이며 어떤 정점에서 시작해 다음 깊이의 정점으로 이동하기전 현재 깊이의 모든 정점을 탐색하며 방문한 정점은 다시 방문하지 않는 알고리즘입니다.
      같은 가중치(같은 거리)를 가진 그래프에서 최단거리 알고리즘으로 쓰입니다.
    - 메모리를 더 씀. 가중치가 같은 그래프내에서 최단거리를 구할 수 있음. 코드가 더 김.

## 3️⃣ 트리 (Tree)

### 📘 정의
- 무방향 그래프
- 사이클이 없는 연결 그래프(Acyclic Connected Graph)
- **계층적 구조(Hierarchical Structure)**

### 📗 주요 용어
- **노드(Node)** : 트리의 기본 단위.
- **루트(Root)** : 트리의 최상단 시작 노드.
- **간선(Edge)** : 부모와 자식을 연결하는 선.
- **부모(Parent)** : 특정 노드의 상위 노드.
- **자식(Child)** : 특정 노드의 하위 노드.
- **리프(Leaf)** : 자식이 없는 끝 노드.
- **깊이(Depth)** : 루트에서 특정 노드까지의 거리(간선 수).
- **높이(Height)** : 트리에서 가장 깊은 노드의 깊이 값.
- **형제(Sibling)** : 같은 부모를 공유하는 노드들.
- 특징 : V - 1 = E (간선 수 = 노드 수 - 1)
- 임의의 두 노드 사이의 경로는 반드시 있으며 하나밖에 없다.
- 내부노드 : 루트노드와 리프노드 사이에 있는 노드를 뜻한다.
- 레벨 : 주어지는 문제마다 다르다.(ex. 루트노드가 0레벨, 루트노드가 1레벨 ...)
- 서브트리 : 트리 내의 부분집합을 의미한다.
- 숲(forest) : 트리로 이루어진 집합.

### 📙 트리 종류
- **이진 트리(Binary Tree)** : 각 노드가 최대 두 개의 자식을 가짐.
    - 정이진 트리(full binary tree) : 자식 노드가 0 또는 2개인 이진 트리를 의미합니다.
      마지막 레벨을 제외하고는 모든 레벨이 완전히 채워져 있으며 마지막 레벨의 경우 왼쪽부터 채워져 있습니다.
    - 변질 이진 트리(degenerate binary tree) : 자식 노드가 하나밖에 없는 이진 트리를 의미합니다.
    - 포화 이진 트리(perfect binary tree) : 모든 노드가 꽉 차 있는 이진 트리를 의미합니다.
- **이진 탐색 트리(BST)** : 왼쪽은 작은 값, 오른쪽은 큰 값으로 정렬.
    - 시간복잡도 : 균형잡히게 분포가 되었다면 탐색,삽입,삭제,수정 모두 O(logN)
    - 균형잡히지 않은것은 선형적으로 분포가 되었다거나 했을때를 의미한다.
- **완전 이진 트리(Complete Binary Tree)** : 왼쪽부터 빈 곳 없이 채워진 형태.
- **균형 트리(Balanced Tree)** : 왼쪽과 오른쪽 높이 차이가 작음.
    - 레드블랙트리

### 📒 순회(Traversal)
- **전위 순회(Preorder)** : Root → Left → Right
- **중위 순회(Inorder)** : Left → Root → Right
- **후위 순회(Postorder)** : Left → Right → Root

## 4️⃣ 세그먼트 트리 (Segment Tree)

### 📘 정의
배열의 구간 정보를 빠르게 계산하기 위한 트리 구조.  
(구간 합, 최소/최대값, 구간 갱신 등 빠른 연산 가능)

### 📗 구성요소
- **리프 노드(Leaf Node)** : 배열의 각 원소 저장.
- **내부 노드(Internal Node)** : 자식 노드들의 결과(합/최소/최대 등) 저장.

### 📙 주요 연산
- **쿼리(Query)** : 특정 구간의 값을 계산하는 연산 (예: 2~5 구간 합).
- **업데이트(Update)** : 특정 원소 변경 시 관련 구간 값 재계산.
- **지연 전파(Lazy Propagation)** : 구간 갱신 시 불필요한 노드 계산을 미루는 최적화 기법.

### 📒 시간 복잡도
- **트리 구축(Build)** : O(N)
- **쿼리(Query)** : O(logN)
- **업데이트(Update)** : O(logN)

### 📘 비교 개념
- **Fenwick Tree (Binary Indexed Tree, BIT)**
    - 세그먼트 트리보다 구조 단순.
    - 구간 합(Range Sum) 문제에 특화.
